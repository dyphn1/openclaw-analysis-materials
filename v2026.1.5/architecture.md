# OpenClaw v2026.1.5 架構說明

## 整體架構

OpenClaw 是一個模組化的智能助手平台，主要由以下部分組成：

1. **核心框架 (Core Framework)**：提供基本的執行環境、工具調用、記憶體管理等。
2. **通道插件 (Channel Extensions)**：支援不同的通訊平台，如 Discord、Telegram、Slack 等。
3. **技能系統 (Skills System)**：透過技能 (Skill) 擴展功能，每個技能是一個獨立的功能模組。
4. **代理人 (Agents)**：負責執行具體任務的代理人，可以是子代理人或 ACP 會話。
5. **設定與配置 (Configuration)**：透過設定檔管理行為和整合方式。

## 模組結構

- `src/`：TypeScript 原始碼，包含核心邏輯。
- `extensions/`：各通道插件的實作。
- `packages/`：共用的 npm 套件。
- `ui/`：前端介面（若適用）。

## 資料流程

1. 使用者透過通道 (如 Discord) 發送訊息。
2. 通道插件將訊息傳遞給核心框架。
3. 核心框架根據訊息內容和設定，選擇適當的技能或代理人進行處理。
4. 處理結果透過同樣的通道插件回傳給使用者。

## 配置擴展性

透過 `packages/` 和 `extensions/`，開發者可以輕鬆加入新的功能或支援新的通訊平台，而不需要修改核心代碼。
